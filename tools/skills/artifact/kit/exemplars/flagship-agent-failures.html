The running incident · INC-4412

A checkout agent at a mid-size fintech looped a bad rollback for 41 minutes. It appears in this file six times — as a context-rot case, an eval blind spot, a coordination failure, and a guardrail data point. One concrete incident keeps four abstract findings honest.

PART 1 / FINDINGS

What 52 sources actually agree on

Most production failures are context-management failures, not model failures.

High confidence
19/52 SOURCES · STRONGEST: INCIDENT POSTMORTEMS (11) · REPLICATED ACROSS ALL TYPES

Teams that instrumented their agents traced most bad outputs to stale, truncated, or contradictory context — the model behaved reasonably given what it was shown. In INC-4412, the fix wasn't a better model: it was deleting 40k tokens of accumulated tool output that contradicted current state.

Eval suites overestimate reliability because they score tasks, not sessions.

Moderate
11 SOURCES · MOSTLY ACADEMIC · THIN INDUSTRY REPLICATION

Benchmarks measure single well-posed tasks; production work is a session where goals drift and errors compound. The agent behind INC-4412 passed its eval suite at 91% the week it shipped. Part 2 shows what the missing 37 points look like.

EVIDENCE · 2 OF 11

"Per-task pass rate was 91%. Per-session — the unit users experience — it was 54%."

Multi-agent systems fail through coordination, and the failure is usually silent.

Moderate
9 SOURCES · ALL POST-2024 · PHENOMENON IS YOUNG

Sub-agents don't crash; they quietly diverge — duplicated work, dropped handoffs, one agent undoing another. INC-4412's 41-minute loop was two writers alternately "fixing" the same config: each local transcript looked normal; only the composed result was wrong.

EVIDENCE · 2 OF 9

"Two writer agents each 'fixed' the config, alternating, 14 times. Neither log looked abnormal in isolation."

POSTMORTEMINC-4412 orchestration write-up · 2026REAL

Coordination overhead grew faster than fan-out gains beyond four concurrent agents on interdependent tasks.

PART 6 / VERDICT

What this file would sign its name to

✓ Fix the scaffolding first

Context discipline and tool-error design have better evidence behind them than model upgrades, across every source type in the file. If you own an agent in production, this is the budget line.

✓ Evaluate sessions, not tasks

The 91→54 gap reproduced in every corpus that measured both. A green task benchmark is necessary and radically insufficient.

✕ Don't universalize guardrails

The heavy-review verdict flips with org size. Any recommendation that omits team scale is overclaiming what the sources support.

✕ Don't trust the multi-agent numbers yet

Nine sources, all post-2024, zero cross-replication — the dashed node in Part 4. Treat finding 3 as a strong hypothesis, not settled fact.
